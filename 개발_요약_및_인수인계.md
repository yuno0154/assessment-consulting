## 4. 항목별 판정 기준 (핵심 포인트)

### 항목 2 — 수행평가명 적절성
- **적절**: 평가 요소(무엇을) + 평가 방법(어떻게) 모두 포함
- **보완 권고**: 평가 방법은 있으나 평가 요소 불명확
- **보완 요청**: 평가 요소 자체가 없거나 전혀 드러나지 않음
- 현장에서 "둘 다 없는 경우는 없고 평가 요소가 빠지는 경우가 많음"

### 항목 3+4+7+6 — 정합성 ★ 가장 중요
**보완 권고 유형 (1가지만 해당 시)**
- 유형 B: 성취수준과 평가요소의 연결 미흡
- 유형 C: 평가요소와 채점기준의 불일치
- 유형 E: 성취수준 진술(질적)과 채점기준 척도(양적) 불일치 ← 가장 빈번
- 유형 G: 질적 평가요소를 양적 기준으로 처리

**보완 요청 유형 (하나 이상 해당 시)**
- 유형 B+C+E+G 복합 (2가지 이상)
- 유형 F: 채점기준 추상적 표현 (채점 불가능)
- 유형 H: 총체적 채점기준 사용 (전면 재설계 필요)
- 유형 I: 수행과제와 성취기준 불일치 ← 심각
- 유형 J: 성취기준 과다 나열 및 무관한 평가 ← 심각

### 항목 8 — 암기형·과제형·AI ★ 최우선 검토
**판단 근거**
- 암기형: 교과서 개념·공식 그대로 재현
- 서·논술형 암기형: 수행 과정 없이 지식만 서술
- 과제형 판단 기준 2가지만 사용 (나머지는 확인 불가):
  - ① 자료 조사·정리·글쓰기 등 수업 중 완성 어려운 요소 포함
  - ② **평가계획서에 제출 기한 명시** ← 핵심 판단 근거
- AI 대리 수행: AI 활용 조건·제한·유의사항 미명시
- 채점기준 암기 유도: 정답 개수 명시, 교과서 내용 그대로 서술 요구

**마무리 공통 문구 (emphasis=true 표시)**
> "위 내용은 제출된 평가계획서의 서술 방식에 근거한 검토 의견으로, 실제 평가 의도와 다를 수 있습니다. 담당 교사께서는 암기형·과제형 평가로 의심되는 부분이 해소될 수 있도록 평가 계획을 수정·보완하여 주시고, 학교에서는 해당 교사와 충분히 협의하신 후 실제 평가 시행 시 모든 학생이 공정하게 평가받을 수 있도록 운영하여 주시기 바랍니다."

### 항목 5 — 미응시 기본점수
- 미응시·미참여 학생: 0점 처리 (기본점수 부여 불가)
- 참여했으나 최저 수행수준 미달 학생: 기본점수 부여 가능
- 두 상황의 처리 기준이 **반드시 구분**되어야 함

## 8. 주요 기술적 포인트 (개발 시 주의사항)

### 한글 인코딩 처리
```javascript
// ❌ 잘못된 방법 (한글 깨짐)
const content = atob(data.content.replace(/\n/g, ''));

// ✅ 올바른 방법
const base64 = data.content.replace(/\n/g, '');
const binary = atob(base64);
const bytes = new Uint8Array(binary.length);
for (let i = 0; i < binary.length; i++) bytes[i] = binary.charCodeAt(i);
const content = new TextDecoder('utf-8').decode(bytes);
```

### 조사 자동 처리
```javascript
function j은는(word) {
  const code = word.charCodeAt(word.length - 1) - 0xAC00;
  return (code >= 0 && code % 28 !== 0) ? '은' : '는';
}
function j이가(word) {
  const code = word.charCodeAt(word.length - 1) - 0xAC00;
  return (code >= 0 && code % 28 !== 0) ? '이' : '가';
}
// 사용: `${perfName}${j은는(perfName)} `
// library.json에는 이(가) 형태로 표기
```

### 수행평가 개수 처리
- 한 과목에 수행평가 2~4개
- 항목 1, 1-1: 과목 단위 1회 판정
